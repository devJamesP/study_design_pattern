CD 커버 가상 프록시를 만든다고 하면
이때 프록시는 원격 객체를 대신하는 용도로 쓰인게 아니라
생성하는 데 많은 비용이 필요한 객체를 숨기기 위한 용도로 쓰였다.
(아이콘용 데이터를 네트워크를 통해서 가져와야 하기 대문에 시간이 많이 걸릴 수 있다.)

[ImageProxy작동 원리]
1. ImageProxy에서는 우선 ImageIcon을 생성하고 네트워크 URL로부터 이미지를 불러온다.
2. 이미지를 가져오는 동안에는 "Loading CD cover, please wait..."라는 메시지를 화면에 표시한다.
3. 이미지 로딩이 끝나면 paintIcon(), getWidth(), getHeight()를 비롯한 모든 메소드 호출을 이미지 아이콘 객체한테 넘긴다.
4.  사용자가 새로운 이미지를 요청하면 프록시를 새로 만들고 위의 과정을 새로 진행한다.
